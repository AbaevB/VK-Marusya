# Задание

Требуется реализовать функционал формы поиска в хедере приложения. *Дополнительно:* улучшить функционал для мобильных окон и блка рейтинга

## Правила проекта

### Стандарты кода

- Строгая типизация TypeScript
- Не создавть блоки стилей в файлах vue. Все стили уже созданы и подключены глобально через `src/assets/scss/style.scss`. При необходимости можно редактировать файлы  `src/assets/scss/components/_header.scss` и `src/assets/scss/components/_search.scss` а так же создавать новые файлы стилей в папке `src/assets/scss/components`
- При стилизации **использовать старые стандарты SASS** - с использованием @import
- Использование БЭМ для CSS селекторов обязательно.
- Код доллжен быть человекопонятным, удобочитаемым, с необходимыми комментариями.

### Архитектура

Использовать имеющиеся соответствующие папки для компонентов, тестов, моков, API, стилей, e.t.c

### Тестирование

- Создать unit-тесты для нового функционала поиска и провести тестирование.
- Минимальное покрытие: 80%

### Git

- Работа выполняется на текущей ветке *feature/search*, используется Git Flow.

- Желательно делать коммиты по стандартам **Conventional Commits**

- Не сливать рабочую ветку *feature/search* в develop и не выполнять feature finish - просто выполнить всю работу на *feature/search*

- По окончании всей работы обновить документацию в отдельном коммите.

## Описание задания

При нажатии на поле «Поиск» (или на кнопку поиск - в мобильном разрешении) Открывается модальное окно с формой поиска под полем ввода поиска должен открываться выпадающий список с результатами поиска - при вводе текста в поле запрашивается список фильмов и отображается 5 первых из полученного списка. Элемент списка содержит миниатюру постера, рейтинг, год выхода, жанр, продолжительность и название фильма. При клике на элемент открывается страница соответствующего фильма.

### Редактирование файла `src/components/blocks/Header.vue`

Удалить из верстки хедера элемент с классом `header__search`, и заменить его блком из файла `src/components/blocks/Search.vue`. Отредактировать файл `src/assets/scss/components/_header.scss`, удалив лишние стили.

### Стили

Вынести все цвета, использыемые в `src/assets/scss/components/_search.scss` в переменные, и сохранить их в файле с остальными глобальными CSS переменными `src/assets/scss/base/_variables.scss`

### Адаптивность

Блок search должен быть адаптивным - на мобильном разрешении поле ввода должно быть скрыто, и должна появляться кнопка, открывающая модальное окно поиска. Для этого нужно использовать БЭМ модификаторы.

### Блок рейтинга

Фоновый цвет к блоку `search__card-rating` должен применяться в зависимости от рейтинга фильма.

### Оптимизация работы с API

Запрос на сервер должен поступать лишь после ввода первыз трех символов в поле поиска

### Дополнительно

#### Блок рейтинга в других секциях

Применить ту же логику изменения фонового цвета к блокам рейтинга в секциях random-film и film.

#### Модальное окно

При открытии модального окна к тегу `body` должен применяться класс `no-scroll`. Это же правило должно быть применено и к уже имеющимся модальным окнам.
