# KODA.md — Инструкционный контекст для проекта Marusia

## Обзор проекта



### Технологический стек

- **Фронтенд**: Vue 3 (Composition API с `<script setup>`)
- **Язык**: TypeScript
- **Сборщик**: Vite
- **Менеджер состояния**: Pinia
- **Маршрутизация**: Vue Router
- **Стили**: SCSS с модульной структурой
- **HTTP-клиент**: Axios (готовность к API) 

### Архитектура

Проект следует стандартной структуре Vue 3 приложения:

- `src/` — исходный код приложения
  - `main.ts` — точка входа, инициализация Vue приложения, подключение Pinia и роутера
  - `App.vue` — корневой компонент с шапкой и навигацией
  - `components/` — переиспользуемые компоненты
    - `blocks/` — блоки интерфейса (Header, FilmCard)
    - `sections/` — секции страниц (RandomFilm, Top-10)
    - `modals/` — модальные окна (AuthModal)
  - `views/` — страницы приложения (HomeView, GenresView, GenreView, FilmView, AccountView, AboutView)
  - `router/` — конфигурация маршрутизации (index.ts)
  - `stores/` — хранилища состояния Pinia (user.ts, films.ts, favorites.ts)
  - `api/` — слой взаимодействия с API (index.ts, auth.ts, films.ts, genres.ts)
  - `assets/` — статические ресурсы (SCSS стили, изображения)
- `public/` — публичные статические файлы (изображения, спрайты, логотипы)
- Конфигурационные файлы: `vite.config.ts`, `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json`, `env.d.ts`, `package.json`

### Алиасы импортов

В проекте настроен алиас `@` для пути `./src`, что позволяет использовать абсолютные импорты вида `@/components/HelloWorld.vue`.

### Работа с API

Проект использует реальное API, доступное по адресу `https://cinemaguide.skillbox.cc/`. Все запросы отправляются через Axios с включёнными cookies (`withCredentials: true`) для поддержки сессионной авторизации.

**Важные моменты:**
- Авторизация работает через сессии с использованием cookies
- При ошибках 401, 404, 500 срабатывают интерцепторы для обработки
- Все API вызовы централизованы в директории `src/api/`
- Хранилища Pinia (`src/stores/`) управляют состоянием и обработкой ошибок
- Компоненты отображают состояние загрузки и ошибки для пользователя

**Основные эндпоинты:**

- `GET /movie` — список фильмов
- `GET /movie/random` — случайный фильм
- `GET /movie/top10` — топ фильмов по рейтингу
- `GET /movie/{movieId}` — информация о конкретном фильме
- `GET /movie/genres` — список жанров
- `POST /auth/login` — авторизация
- `POST /user` — регистрация
- `GET /profile` — информация о текущем пользователе
- `GET /auth/logout` — выход из системы
- `GET /favorites` — избранные фильмы
- `POST /favorites` — добавление в избранное
- `DELETE /favorites/{movieId}` — удаление из избранного

## Сборка и запуск

### Предварительные требования

- Node.js версии ^20.19.0 или >=22.12.0 (указано в `engines` package.json)
- Менеджер пакетов npm (или другой совместимый)

### Установка зависимостей

```bash
npm install
```

### Разработка с горячей перезагрузкой

```bash
npm run dev
```

Запускает сервер разработки Vite. Приложение будет доступно по умолчанию на `http://localhost:5173` (или другому порту, если 5173 занят).

### Проверка типов

```bash
npm run type-check
```

Запускает `vue-tsc` для статической проверки типов TypeScript в файлах `.vue` и `.ts`.

### Сборка для продакшена

```bash
npm run build
```

Выполняет проверку типов и сборку проекта. Собранные файлы помещаются в директорию `dist/`.

Альтернативно можно собрать без проверки типов:

```bash
npm run build-only
```

### Просмотр собранного приложения

```bash
npm run preview
```

Запускает локальный сервер для просмотра собранного приложения (требует предварительного выполнения `npm run build`).

## Правила разработки

### Стиль кодирования

- **Компоненты Vue**: используются однофайловые компоненты (`.vue`) с синтаксисом `<script setup lang="ts">` для Composition API.
- **TypeScript**: строгая типизация, включена опция `noUncheckedIndexedAccess` для дополнительной безопасности.
- **Именование файлов**: компоненты используют PascalCase (например, `HelloWorld.vue`), хранилища — camelCase (например, `counter.ts`).
- **Импорты**: предпочтительны абсолютные импорты с использованием алиаса `@`.
- **Не использовать** scoped CSS в компонентах - все стили заданы в файле `src/assets/scss/style.scss`, и в подключенных к нему файлах

### Структура состояния

Состояние управляется через Pinia с использованием трёх основных хранилищ:
- `user.ts` — управление авторизацией и данными пользователя
- `films.ts` — работа с фильмами (случайный фильм, топ-10, фильтрация по жанрам, поиск)
- `favorites.ts` — управление избранными фильмами

Каждое хранилище включает:
- Реактивные переменные состояния (`ref`)
- Вычисляемые свойства (`computed`)
- Асинхронные методы для работы с API
- Обработку ошибок и состояний загрузки

### Маршрутизация

Маршруты определяются в `src/router/index.ts`. Используется `createWebHistory` для HTML5 History Mode. Реализованы маршруты:
- `/` — главная страница (случайный фильм + топ-10)
- `/genres` — список жанров
- `/genre/:genre` — фильмы по конкретному жанру
- `/film/:id` — детальная страница фильма
- `/account` — личный кабинет (требует авторизации)
- `/about` — о проекте (ленивая загрузка)

### Тестирование

Необходимо включить `vitest` для тестирования, и покрыть тестами 
### Инструменты разработки

- Рекомендуемая IDE: VS Code с расширением [Volar](https://marketplace.visualstudio.com/items?itemName=Vue.volar) (Vetur должен быть отключён).
- Для отладки в браузере рекомендуется установить Vue.js Devtools.
- В проекте уже подключен плагин `vite-plugin-vue-devtools` для интеграции инструментов разработки Vue.

### Конфигурация Vite

Конфигурация сборщика находится в `vite.config.ts`. Подключены плагины:
- `@vitejs/plugin-vue` — поддержка `.vue` файлов
- `@vitejs/plugin-vue-jsx` — поддержка JSX в Vue
- `vite-plugin-vue-devtools` — инструменты разработки

Настроен алиас `@` для пути `./src`.
